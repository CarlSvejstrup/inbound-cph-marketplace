### M2 — Core skills
**Status:** Done (8 skills shipped)

- `client-brief` — synthesised one-page brief
- `proactivity-scan` — three ranked recommendations
- `weekly-pulse` — two-minute status delta
- `voice-check` — severity-ranked draft review
- `onboard` — guided Danish setup writing local CLAUDE.md + guide.docx
- `ads-audit` — full Google Ads paid search audit, outputs HTML slide deck + PDF (v0.4.0)
- `search-terms` — focused Google Ads search terms analysis for one client (v0.6.0). Read-only: pulls `search_term_view` via GAQL (cost-desc, ENABLED campaigns only), aggregates one row per term, rule-classifies into Spild / Vinder / Neutral (dynamic CPA threshold with absolute floor for low-conversion accounts), LLM-classifies the costly-Neutral subset into Irrelevant / Nyt emne, cross-checks winners against existing keywords, and suggests a negative level (ad group / campaign / shared list, labelled a recommendation). Output is a six-tab colour-coded `.xlsx` (Overblik, Spild, Vindere, Irrelevante, Nye emner, Raadata) built by `build-sheet.py` (openpyxl) and uploaded via the Drive connector or saved locally — same xlsx mechanic as rsa-copy, so no `gws`/Sheets API and runs in Cowork. GAQL field shape, micros, grain, and zero-value-account CPA handling verified against a live account (KFH kloak). `build-sheet.py` smoke-tested end-to-end (six coloured tabs, header fills, Spild cost colour-scale). Not yet run against a full real account with the LLM pass.
- `rsa-copy` — landing page to Editor-ready RSA ad-copy sheet (v0.5.0). Builds a fresh `.xlsx` each run from a bundled `template.xlsx` (generated by `build-template.py`, openpyxl) that carries live `=LEN()` formulas + red over-length conditional formatting; `fill-sheet.py` fills only the text cells (15 headlines + 4 descriptions + 2 paths) under Google's hard limits (30/90/15) and validates lengths. Saves to Drive via the connector's `create_file` (user picks folder via `parentId`) OR locally. Runs in Cowork AND locally — no `gws`, no clone, no Sheets API. Verified end-to-end: template build, fill + length-guard, and Drive upload round-trip keeping `=LEN()` live.
